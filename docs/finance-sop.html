一、岗位概述

深圳市锦和科技有限公司财务部负责全公司跨境电商业务的财务核算、资金管理与报表编制。业务平台覆盖 Amazon、TikTok Shop、Walmart、MercadoLibre、Temu 五大平台，收款账户含香港诚谦、国内公账及内账（老板账）。本手册规定财务各岗位的标准操作流程，确保数据准确、流程可追溯、合规无遗漏。

第一步：平台销售数据核对

应收 vs 实收的核心区别 应收（领星 ERP 订单金额）= 平台应支付给卖家的金额，尚未扣除平台佣金、FBA费、汇损等。
实收（飞书多维表格到账记录）= 实际已换汇入账的人民币金额，为最终到手金额。
两者数据来源不同，不可混用，在看板和报表中须分开展示。
  1. 1
    导出领星 ERP 当月订单数据
    登录领星 ERP → 数据中心 → 订单报表 → 选择当月日期范围 → 按平台分别导出。覆盖全部六个平台代码（10001/10008/10011/10012/10022/10024）。MercadoLibre 导出后需将金额 ×50% 再计入汇总。
  2. 2
    与平台后台数据核对（应收核对）
    逐平台登录后台，核对订单总金额：
    • Amazon Seller Central → Reports → Payments → 下载 Settlement Report
    • TikTok Shop 后台 → 财务 → 结算报告
    • Walmart Seller Center → Payments → Statement
    • MercadoLibre → Mis Ventas → Facturación
    • Temu 后台 → 财务中心 → 对账单
    差异超过 5% 必须排查原因（退款未同步、待确认订单、跨月订单等）。
  3. 3
    核对飞书到账记录（实收核对）
    打开飞书多维表格（App Token: Ex71bGF5gaqrtYsZhnxcmEpZnmg），筛选当月「收支=收入」的记录，核对各账户实际到账金额与银行流水是否一致。确认无漏录、无错录。
  4. 4
    记录核对结果
    在月结工作底稿中填写：各平台应收金额（原币）、换算 RMB、实收金额、应收实收差异及原因说明。

第二步：成本归集

成本科目数据来源归集方法
销货成本（COGS）领星 ERP 利润报表 / 采购入库单按平台、按 SKU 归集当月已销商品成本
头程物流费货代发票 / 系统录入按批次分摊到当月，跨月批次按实际到货比例分摊
FBA/平台仓储费Amazon Settlement Report / 领星利润报表直接读取平台结算单中的仓储费用行
平台佣金各平台结算报告直接读取结算单佣金行（Amazon referral fee 等）
广告费Amazon 利润报表 API / 各平台广告后台Amazon 使用 API 实际数据；其他平台手动录入 costs_config.json
固定成本costs_config.json → fixed_costs按月直接计入（工资、房租、软件订阅等）
跨月费用分摊原则 头程物流费按货物抵达目的仓时间归属月份，不按发货时间。如批次跨越两个月到货，按货量比例分摊。仓储费按平台账单周期归属，不得随意调整。
  1. 1
    更新 costs_config.json
    路径：analysis/costs_config.json。确认当月固定成本（工资/房租/SaaS订阅）和广告费（非Amazon平台）已按实际金额填写。Amazon 广告费由 API 自动获取，无需手动填写。
  2. 2
    收集货代/物流发票
    向各货代索取当月发票，核对金额，录入财务系统。头程费需标注对应批次号和平台，用于后续分摊。
  3. 3
    核对仓储和佣金（Amazon 重点）
    下载 Amazon Settlement Report，提取 FBA Storage Fee、Referral Fee、FBA Fulfillment Fee 等费用行，与领星利润报表 API 数据比对，确认一致后录入月结底稿。
  4. 4
    汇总成本归集表
    按平台分列成本科目，加总确认，填入利润表草稿。

第三步：收入确认与汇率换算

  1. 1
    确认当月换汇均价
    向银行/兑换平台获取当月实际换汇明细，计算加权平均换汇率（总人民币入账 ÷ 总外币金额）。将结果更新到 costs_config.json → exchange_rates 对应月份。
  2. 2
    应收金额按统一汇率换算
    领星订单金额（USD/MXN/GBP）× 当月均价汇率 = 应收人民币。MercadoLibre 需先 ×50% 再换算。此金额用于利润表收入行。
  3. 3
    实收金额确认
    飞书多维表格当月收入合计即为实收人民币，直接读取「人民币」字段合计值。此金额用于现金流量表收款项目。
  4. 4
    计算应收实收差异（汇兑损益）
    应收换算值 - 实收金额 = 当月汇兑损益。正值为汇兑收益，负值为汇兑损失，计入利润表其他收益/损失项。

第四步：财务三表生成

  1. 1
    运行财务报表生成脚本
    在服务器执行：cd ~/wsl-workspace/JHFinance && python3 analysis/generate_reports.py。脚本读取 raw-data/ 下的 Excel 文件，优先级：STANDALONE_2026 月度独立表 > A3 USD 报表 > 流水明细 journal。
  2. 2
    生成 HTML 看板
    执行：python3 analysis/generate_html_report.py，生成最新财务看板，并发布到 GitHub Pages（JHFinance-public/index.html）。
  3. 3
    复核利润表关键数字
    检查以下项目是否合理：
    • 综合毛利率是否在合理区间（跨境业务参考：40%-75%）
    • 各平台收入占比是否符合业务预期
    • 广告费/收入比率（Amazon 建议 <5%，TikTok 可更高）
    • MercadoLibre 是否已按 50% 计入
  4. 4
    更新资产负债表
    更新现金及存款余额（三账户实际余额）、应收账款（各平台未结算金额）、存货货值（库存 Excel 当月数）、应付账款（待付供应商款项汇总）。
  5. 5
    编制现金流量表
    经营活动现金流 = 实收客款 - 实付供应商货款 - 实付物流费 - 实付广告费 - 实付员工工资及费用。数据来源：飞书多维表格当月全部收支记录。

报销单填写规范

  1. 1
    填写基本信息
    报销人姓名、部门、报销日期、联系方式。日期填写实际发生日期，不得填写报销提交日期。
  2. 2
    逐笔列明费用
    每笔费用需单独列行，填写：日期、费用类型、发生事由（简明描述业务目的）、金额。不得合并多笔填写。
  3. 3
    粘贴发票
    发票按费用顺序整齐粘贴在 A4 纸背面。增值税发票须核验税号。电子发票下载后打印并附上。
  4. 4
    提交审批
    纸质版送直属主管签字，同时在飞书工作流（或企业微信）提交电子审批流。电子流审批通过后，方可到财务部门领款。

发票要求

每周平台回款核对流程

  1. 1
    下载各平台结算报告（每周一上午）
    逐平台下载上周结算期的结算报告，保存至 raw-data/settlements/YYYY-MM/ 文件夹。文件命名格式：平台名_结算日期_金额.csv。
  2. 2
    核对银行到账记录
    登录香港诚谦网银，检查上周是否有预期到账款项，金额与结算报告是否一致。差异超过 100 美元需排查原因（拒付、费用扣除等）。
  3. 3
    录入飞书多维表格
    确认到账后，在飞书表格中录入：日期、内容（"Amazon 第N期结算"等）、收支="收入"、分类、关联账户、人民币金额（按换汇实际金额）、美金金额。
  4. 4
    跟踪逾期未到账款项
    对比预期到账日期与实际到账，超过正常周期 3 天仍未到账需发邮件/工单询问平台。记录在应收账龄表中。

MercadoLibre 合作方分成结算

MercadoLibre 50% 分成机制 锦和科技与合作方共同运营 MercadoLibre 账号，收入按 50:50 分成。财务处理规则：
• 领星订单数据中 MercadoLibre 的销售额，内部核算时只计 50% 为公司收入
• 每月结算后，将合作方应得 50% 通过银行转账支付给合作方
• 支付时须取得合作方签收确认，并在飞书表格记录为「支出」，分类「合作分成」
• 如合作比例有变更，须书面通知财务部并同步更新 lingxing_api.py 中的 alloc 参数
  1. 1
    每月月结后计算 ML 分成金额
    ML 当月 MXN 销售额 × 汇率 × 50% = 合作方应得金额（人民币）。
  2. 2
    发送分成对账单给合作方
    将计算明细（订单量、MXN金额、汇率、分成比例、人民币金额）发给合作方确认。双方确认无误后执行付款。
  3. 3
    执行打款并存档
    通过国内公账打款，保留转账截图，存档至 raw-data/ml-settlements/。

付款执行 SOP

  1. 1
    核对三单一致性
    付款前核对：采购订单（数量/单价）、收货入库单（已收到货物）、供应商发票（金额/税率）。三单不符不得付款。
  2. 2
    确认供应商账户信息
    首次付款须通过已验证渠道（合同、官方邮件）确认账户信息，不得仅凭电话或微信通知的账户付款。更换账户须重新核验。
  3. 3
    执行打款
    通过国内公账（常规供应商付款）或香港诚谦账户（境外供应商/货代 USD 付款）执行转账。截图留存。
  4. 4
    录入飞书表格
    录入字段：日期、内容（"付XX公司货款-第N批"）、收支="支出"、分类（货款/物流费/广告费等）、关联账户、人民币金额。
  5. 5
    通知供应商并索取收款确认
    付款后微信/邮件通知供应商，请对方确认到账。将确认截图存入 raw-data/payments/YYYY-MM/。
付款安全红线 • 绝对禁止：仅凭微信/电话通知更改付款账户，不核实直接付款
• 绝对禁止：无审批单据的紧急付款（即使老板口头同意，也须在飞书补录审批）
• 绝对禁止：向个人账户付货款（须为公司对公账户，特殊情况须有书面说明）
• 防范骗局：收到"供应商换账号"通知，必须电话回拨至之前已知号码二次确认

货代/头程物流付款要点

  1. 1
    核对运费清单
    货代发票须含：批次号/提单号、起运/目的地、重量体积、运费单价、总金额。与发货单核对。
  2. 2
    区分记账月份
    按货物到达目的仓时间归属月份记账（非发货时间）。跨月批次须拆分。
  3. 3
    USD 货代付款
    通过香港诚谦账户发送 USD 电汇，需提前确认受益人 SWIFT/IBAN 信息，留存 SWIFT 汇款底单。

换汇操作 SOP

  1. 1
    汇率评估
    换汇前参考当日银行中间价，与过去 30 天平均汇率对比。如汇率比月均高 0.5% 以上，可适当提前换汇。不建议预测汇率走势，优先保证资金流转顺畅。
  2. 2
    执行换汇
    登录香港诚谦网银/柜台，操作外汇兑换，记录实际成交汇率和金额。截图留存。
  3. 3
    更新汇率配置
    每笔换汇后，将实际换汇汇率更新到月度换汇汇率跟踪表。月底计算当月加权平均换汇率，填入 analysis/costs_config.json → exchange_rates 对应月份。
  4. 4
    录入飞书表格
    换汇到账后在飞书录入：日期、内容（"USD换汇到账 XXUSD@7.XX"）、收支="收入"、分类="换汇"、关联账户="国内公账"、人民币金额、美金金额。

汇兑损益计算

月度汇兑损益核算方法 账面应收（领星订单金额 × 月初汇率记账价）与实际换汇到账（实收人民币）之间的差额，即为当月汇兑损益。
• 实收 > 账面应收 = 汇兑收益（计入其他收益）
• 实收 < 账面应收 = 汇兑损失（计入财务费用）
跨境业务每月均有汇兑损益，属正常现象。若当月损失超过收入 3%，须向老板说明原因。

十、应急处理

场景一：平台回款长期未到账

  1. 1
    确认结算报告
    登录平台后台，确认该期结算报告已生成，核对金额和状态是否为"已结算"。
  2. 2
    检查账户信息
    确认平台绑定的收款账户是否正确，是否有账户信息变更未同步。
  3. 3
    提交平台工单
    通过平台官方渠道（卖家支持中心）提交查询工单，附上结算报告截图和银行账号信息，要求平台核查。
  4. 4
    上报主管
    超过正常回款周期 7 天未到账，立即上报财务主管和老板，必要时寻求法律/平台政策支持。

场景二：发现异常付款（疑似诈骗）

  1. 1
    立即联系银行止付
    发现错误付款后，第一时间电话联系开户银行，申请紧急止付或撤回转账（境内转账有时可撤回）。
  2. 2
    保存全部证据
    截图保存：聊天记录、付款凭证、银行流水、对方账号信息。不得删除任何相关记录。
  3. 3
    立即上报老板
    任何金额的疑似诈骗付款都必须第一时间上报，不得隐瞒或自行处理。
  4. 4
    向公安机关报案
    金额较大（5,000 元以上）的诈骗付款，须向当地公安机关报案，取得受案回执用于后续追偿。

场景三：看板数据不更新 / 脚本报错

  1. 1
    查看更新日志
    执行：tail -50 ~/wsl-workspace/JHFinance/daily_update.log，查看最近一次运行是否报错。
  2. 2
    检查 API Token 是否过期
    飞书 API Token 有效期 2 小时（自动刷新），领星 API Token 有效期较长。如报 401 错误，检查 analysis/feishu_api.py 或 lingxing_api.py 中的 App ID/Secret 是否有效。
  3. 3
    手动运行脚本
    执行 cd ~/wsl-workspace/JHFinance && bash analysis/daily_update.sh 手动触发更新，观察实时报错信息。
  4. 4
    联系技术支持
    若无法自行排查，截图报错信息，联系系统维护人员处理。在脚本修复前，可手动维护 Excel 原始数据保证月结不中断。

场景四：月结数据异常（利润率偏差过大）

  1. 1
    逐项排查收入端
    确认 MercadoLibre 是否已按 50% 计入；确认汇率是否使用当月均价；确认无平台重复计算。
  2. 2
    逐项排查成本端
    确认 COGS 是否按已销数量计算（非采购数量）；确认头程费是否按到货月份归属；确认广告费未重复录入（API + 手工双重计入）。
  3. 3
    与运营核实业务情况
    毛利率大幅下降可能有正当原因（清货促销、大额广告投放、新品冷启动），需与运营团队核实业务背景后再判断是否为数据错误。
  4. 4
    修正并记录
    找到错误原因后，修正数据，在月结工作底稿中记录"发现问题 → 原因 → 修正方式"，防止下月重复发生。
